1. Blood Donation Application

Problem and Requirements -

  1. Increasing demand for blood samples to support Drug Research.
  2. Blood samples from donors outside company costed £1000+ per sample due to costs like advertising, transport etc.
  3. Asking company employees to donate blood costed much less (£50 per donor including transport, health center fees etc.).
  4. Needed a flexible and efficient system to manage the blood donation process and track blood samples.

Solution -

  1. Developed a lightweight web application. Backend – Flask, Database – MySQL, Frontend – Bootstrap.
  2. Application features -
    1. Automatically selected potential donors based on business rules.
    2. Emailed respective people(donors, admins, health center staff etc.) with information and calendar invites.
    3. Generated barcode labels for tubes.
    4. Request and donor reports.
    5. Request tracking using statuses and donor tracking using statuses/barcodes.
    6. Donor Approval Mechanism.

Also, helps make the app maintainable as just the code for the web application needs to be updated when a change is needed.

3. IoT application

Requirements -

  1. Chemists needed an application that would allow them to remotely collect and visualize data from some of the instruments in the lab to monitor the progress/status of their experiments.

Solution -

  1. Connected a Raspberry Pi 3 to each of the lab equipment that needed monitoring and put it on the network.
  2. Created a web application that fired commands at the machine through the Raspberry Pi and grabbed data in return.
  3. Chemists could set a schedule and an interval (for each data point) at the start of the experiment. The web application would fire commands at the instrument and grab data at the specified time intervals, till it reached the end of the schedule.
  4. Real-time visualizations of multiple data points using High Charts.
  5. Detailed custom reports for captured data.
